Comment on the advantages and disadvantages of using a Karnaugh map to simplify a logical expression instead of Boolean algebra.

So essentially, there is a difference between "simplest form" and "simplest sum of products form". The latter is what was desired, and is preferable for a couple of reasons. Usually sum-of-products is preferable when communicating a design (since it separates the states out) or aligning a design with some (general purpose) implementation technology (e..g, a PLA). Also, can be useful when using logic components that have greater than binary input (eg. tri-OR gates in this example) or keeping the spacing between components more balanced in a chip.

tl;dr, karnaugh map always gets the simplest sum of products, but not necessarily the simplest form (the fewest number of logic gates)
